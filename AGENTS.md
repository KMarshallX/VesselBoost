# AGENTS.md

## Project Overview

**Project name:** VesselBoost for T2*-weighted MRI vessel segmentation
**Project type:** Research codebase for developing and evaluating vessel segmentation methods
**Primary domain:** Brain vasculature segmentation in MRI  
**Primary language:** Python  

This VesselBoost project was intially developed for TOF MRA vessel segmentation. The pretrained models were trained on 14 TOF MRA volumes from the StudyForrest (https://studyforrest.org/) dataset. The 14 volumes have corresponding manual segmentations of the brain vasculature, as well as two sets of segmentations generated by a morphological Frangi filter-based pipeline (one contains more false positives, one contains more false negatives). The three sets of segmentations were used as the training targets for the VesselBoost models, which were trained to learn the mapping from TOF MRA images to vessel segmentations.

In this development branch, the focus is on extending the VesselBoost pipeline to T2*-weighted MRI, which has different contrast and vessel appearance than TOF MRA. The goal is to develop a segmentation method that can handle the challenges of T2*-weighted MRI, such as low contrast and small vessel size, while leveraging the existing VesselBoost framework and pretrained models as much as possible.

## Project Priorities:

1. **Automatic Preprocessing of Three-Channel Input**
- Automatic preprocessing pipeline with one T2* MRI input and three-channel output (inverted T2* MRI, vesselness-based tubularity map, projection-based continuity prior)

2. **Candidate Vessel Evidence Generation**
- Three complementary vessel candidates would be generated from the three-channel input. The three candidates are:
  - **Intensity-based candidate:** Prediction from the current pretrained VesselBoost model on the inverted T2* MRI channel, which captures vessel structures based on intensity contrast. - The pretrained model contributes a learned vessel prior but is affected by domain shift.
  - **Vesselness-based candidate:** Generated by thresholding the vesselness-based tubularity map (e.g., Frangi filter response) to capture tubular structures that may correspond to vessels. - The vesselness map contributes local tubular geometry but may over-detect non-vascular ridges.
  - **Continuity-based candidate:** Generated by thresholding the projection-based continuity prior channel (e.g., minimum intensity projection along vessel trajectories) to capture continuous vessel segments. - The continuity prior may improve connectivity but may merge nearby dark structures.

3. **Candidate Fusion and Refinement**
- Based on the current stage output, the initial segmentation (intensity-based candidate) contains substantial false positives and false negatives. The vesselness-based and continuity-based candidates can provide complementary evidence to refine the initial segmentation. The fusion strategy may involve:
  - a high-confidence seed region defined by voxels supported by at least two of the three candidates 
  - a broader candidate region will be defined from the union of all candidates, constrained by a vesselness floor
  - small isolated components will be removed
  - may incorporate an optional shape-based cleanup to suppress obviously non-tubular structures (e.g., large blobs, thin sheets)

4. **Model Adaptation**
- The fused proxy label would then be used to adapt VesselBoost models to T2* MRI domain, which may involve:
  - fine-tuning the pretrained model on the fused proxy labels (TTA style adaptation)
  - training a new model from scratch using the fused proxy labels (Booster module)
  - exploring domain adaptation techniques (e.g., adversarial training, self-supervised learning) to better align the feature representations between TOF MRA and T2* MRI - this is not the current priority but may be explored if the above steps do not yield satisfactory results.

5. **Postprocessing**
- The final segmentation may be further refined by postprocessing steps such as:
  - conservative thresholding to preserve small vessels
  - removal of very small isolated componets
  - (OPTIONAL) short-gap reconnection only when geometrically plausible (e.g., based on vesselness and continuity evidence)

## Core Technical Decisions
- Keep the current code structure and workflow as much as possible to maintain consistency with the original VesselBoost pipeline. 
- The new components for T2* MRI adaptation should be integrated into the existing framework rather than creating a separate codebase.
- Prefer simple and interpretable implementations over highly optimizd ones for early development stages.
- Prioritize consistency and reproducibility and safe validation over conveniece.

## File and Module Organization
Keep modules small and single-purpose

Current file structure of the VesselBoost repository is as follows:

- library/: utility functions and classes for data loading, augmentation, evaluation, etc.
- config/: configuration files for different working pipeline (prediction, tt-adaptation, training, etc.)
- documentation/: documentation files (instructions, design docs, etc.)
- models/: DL model architectures
- saved_models/: pretrained model weights
